## Setup

Here’s the system requirements for this tutorial:

- [Node.js](https://nodejs.org/) 10 or later
- Your favorite text editor and terminal

macOS, Windows (including WSL), and Linux are all supported.

> If you are on Windows, try to use PowerShell or Git Bash (Comes with [git](https://git-scm.com/downloads)). Here’s why: Next.js works with any shell or terminal, but we'll be using some UNIX-specific commands in this guide.

### Step 1: Create files

First, we will:

- Create and move into the `hello-next` directory
- Use `npm` to create `package.json` and install `react`, `react-dom`, and `next`
- Create `index.js` under the `pages` directory

Run these commands in order:

```bash
mkdir hello-next
cd hello-next
npm init -y
npm install --save react react-dom next
mkdir pages
touch pages/index.js
```

> If you prefer [Yarn](https://yarnpkg.com/), you can use `yarn add` instead of `npm install --save`.

### Step 2: Create a page component

Next, open `index.js` in the `pages` directory inside `hello-next` and add the following:

```jsx
const Index = () => <h1>Hello Next.js</h1>;

export default Index;
```

This is a simple React component that displays <strong>Hello Next.js</strong> in `<h1>` tag.

### Step 3: Set up scripts

Finally, open `package.json` in the `hello-next` directory and replace the contents of `scripts` with the following:

```json
"scripts": {
  "dev": "next",
  "build": "next build",
  "start": "next start"
},
```

This adds the following commands:

- `npm run dev`: Start the development server. You’ll use this often!
- `npm run build`: Build the project for production.
- `npm run start`: Start the production server.

### Running Next.js

Run the following command to start the development server:

```bash
npm run dev
```

Then open [http://localhost:3000](http://localhost:3000/) from your browser.
